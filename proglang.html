Chapter 6: Programming Languages (through page 340)
TERM DEFINITION
Machine language machine languages are programs for modern computers that consist of sequences of instructions that are encoded as numeric digits.
Debugging tedious task of locating and correcting errors before a job is finished.
assembler programs developed to convert mnemonic expressions into machine language instructions.
assembly language A mnemonic system for representing programs. They represented a giant step foward in the search for better programming techniques.
translator a program written that translated programs expressed in these high-level primitives into machine-language programs.
compiler these transalators programs were often called compilers sinc e they often had to compile several machine instructions into short sequences to stimulate the activity requested by a single high-level primitive.
interpreter emerged as another means of implementing third-generation languages. These programs were similar to translators except that they executed the instructions as they were translated instead of recording the translated version for future use.
cross-platform software software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout the network.
natural language languages precisely defined by grammers (such as English, German, and Latin).
formal language language that evolved over time without formal grammatical analysis (such as programming languages).
machine independence
high-level programming language
object-oriented programming (OOP) a software system is viewed as a collection of units, called objects, each of which is capable of perfoming the actions that are immediately related to itself as well as requesting actions of other objects.
scripting language these languages are typically used to perform administrative tasks rather than to develop complex programs.
data types:
  • integer
  • float
  • string
  • the type integer refers to numeric data consisting of whole numbers, probably stored using two's complement notation.
  • the type float refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
  • the type string refers to string of characters.
primitive data type the data types that are included as primitives in a programming language such as int fot integer and char for character.
data structure the conceptual shape or arrangment of data.
Python collection data structure
  • List
  • tuple
  • set
  • dictionary
  • lists are used to store multiple items in a single variable, created using square brackets.
  • a tuple is a collection which is ordered and unchangeable, it is written with round brackets.
  • a set is a collection which is unordered, unchangeable, and unindexed, it is written with curly brackets.
  • a dictionary is a collection which is ordered, changeable and do not allow duplicated, it is written with curly brackets, and have keys and values.
function a set of instructions for performing a task that can be used as an abstract tool by other program units.
local variable a variable declared within a function.
global variable variables whose scopes are not restricted to a particular part of a program.
source program the program in its original form.
object program the translated version of the program.
